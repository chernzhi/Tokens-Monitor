# AI Token 监控客户端 — 员工使用手册

> 版本 2.2.0 | 适用于 Windows 系统

## 一、这是什么？

AI Token 监控客户端（ai-monitor）是公司内部的 AI 用量统计工具。它在你的电脑本地运行，**自动记录**你使用各种 AI 开发工具（如 Cursor、VS Code Copilot、Claude Code 等）时产生的 Token 消耗量，并上报到公司统计服务器。

**它不会：**
- 读取或上传你的代码内容
- 影响 AI 工具的正常使用
- 降低你的网速

**它只做一件事：** 统计每次 AI 请求用了多少 Token。

## 三、下载与安装

### 3.1 下载

浏览器打开以下地址，自动下载 `ai-monitor.exe`：

```
https://otw.tech:59889/api/extension/client
```

将下载的文件放到一个固定目录（如 `C:\Tools\ai-monitor\`）。

### 3.2 首次安装（推荐方式）

**双击 `ai-monitor.exe`**，程序会自动检测到没有配置文件，在浏览器中打开安装向导页面。

按照向导操作：

1. **第一步 — 注册账号**
   - 填写 **姓名**（真实姓名，用于统计报表）
   - 填写 **邮箱**（公司邮箱，作为登录凭证）
   - 填写 **部门**（可选）
   - 设置 **密码**（至少 6 位）
   - 点击「注册」

2. **第二步 — 一键安装**
   - 确认信息无误后点击「一键安装」
   - 程序会自动完成：安装 CA 证书、设置环境变量、注册开机自启

3. **安装完成后，关闭向导页面，重新打开你的 IDE 和终端即可。**

> 如果你已有账号（例如之前用工号注册过），可以在向导中选择「登录」或「绑定已有账号」。

### 3.3 命令行安装（可选）

如果你不想使用网页向导，可以先手动创建 `config.json`，然后执行：

```bash
# 全局安装（推荐）—— 一键配置所有开发工具
ai-monitor.exe --global-install

# 或最简安装 —— 仅安装 CA 证书，不改动系统代理和环境变量
ai-monitor.exe --install
```

## 四、日常使用

### 4.1 全局模式（推荐，安装后无需关心）

通过向导或 `--global-install` 安装后，ai-monitor 会：
- **开机自动启动**（后台运行，无窗口）
- **自动覆盖所有开发工具**（VS Code、Cursor、JetBrains、CLI 工具等）

你只需要正常使用你的开发工具，Token 用量会自动被记录。

### 4.2 定向模式（可选，不改动系统环境）

如果你不想修改系统代理和环境变量，可以用 `--launch` 模式：

```bash
# 启动 Cursor，仅 Cursor 的 AI 流量被监控
ai-monitor.exe --launch-preset cursor

# 启动 VS Code
ai-monitor.exe --launch-preset vscode

# 启动 PowerShell，在里面运行 Claude Code / Aider 等 CLI 工具
ai-monitor.exe --launch-preset powershell

# 启动任意程序
ai-monitor.exe --launch -- "C:\path\to\your\program.exe"
```

**可用预设列表：**

| 预设名 | 说明 |
|--------|------|
| `vscode` | VS Code |
| `cursor` | Cursor |
| `windsurf` | Windsurf |
| `kiro` | Kiro |
| `trae` | Trae |
| `zed` | Zed |
| `idea` | IntelliJ IDEA |
| `webstorm` | WebStorm |
| `pycharm` | PyCharm |
| `goland` | GoLand |
| `powershell` | PowerShell 终端 |
| `cmd` | CMD 终端 |

### 4.3 查看运行状态

ai-monitor 运行后，浏览器访问：

```
http://localhost:18090/status
```

返回 JSON 格式的运行状态，包括已上报条数、Token 总量等。

## 五、配置页面（修改密码、更新配置）

ai-monitor 运行期间，可随时在浏览器访问配置页面：

```
http://localhost:18090/wizard
```

在这里你可以：
- **修改密码** — 选择「修改密码」Tab，填写邮箱、旧密码和新密码
- **忘记密码** — 选择「忘记密码」Tab，通过邮箱 + 姓名验证身份后重置密码
- **重新登录** — 如果 Token 过期，重新登录获取新的认证令牌
- **更新配置** — 修改上报服务器地址、上游代理等设置

> 端口默认为 18090。如果配置文件中修改了端口，请使用实际端口。

## 六、卸载

```bash
# 全局卸载（移除 CA 证书、环境变量、开机自启）
ai-monitor.exe --global-uninstall

# 普通卸载
ai-monitor.exe --uninstall
```

卸载后重新打开终端和 IDE 即可。

## 七、配置文件说明

配置文件 `config.json` 与 `ai-monitor.exe` 在同一目录下，通常由安装向导自动生成。

```json
{
  "server_url": "https://otw.tech:59889",
  "user_name": "张三",
  "user_id": "zhangsan@company.com",
  "department": "公共技术部",
  "port": 18090,
  "upstream_proxy": "",
  "auth_token": "xxx..."
}
```

| 字段 | 说明 | 是否必填 |
|------|------|----------|
| `server_url` | 统计服务器地址 | 自动填写 |
| `user_name` | 你的姓名 | 是 |
| `user_id` | 你的邮箱/工号 | 是 |
| `department` | 部门名称 | 否 |
| `port` | 本地监听端口（默认 18090） | 否 |
| `upstream_proxy` | 上游代理地址（公司需代理上外网时填写） | 否 |
| `auth_token` | 登录后自动获取的认证令牌 | 自动填写 |

## 八、常见问题

### Q: 安装后 IDE 的 AI 功能不能用了？

检查 CA 证书是否安装成功。运行：
```bash
ai-monitor.exe --install
```
如果提示 CA 安装失败，可以手动安装：
```
certutil -addstore -user Root "%APPDATA%\ai-monitor\ai-monitor-ca.crt"
```
然后重启 IDE。

### Q: 公司网络需要代理才能上外网？

在 config.json 中添加 `upstream_proxy` 字段，填写你的代理地址：
```json
{
  "upstream_proxy": "http://proxy.company.com:8080"
}
```
或在安装向导中的「高级选项」里填写。ai-monitor 会自动将非 AI 流量转发给你的公司代理。

### Q: 如何确认 ai-monitor 正在运行？

浏览器访问 `http://localhost:18090/status`，如果返回 JSON 数据则说明运行中。

### Q: 如何修改密码？

浏览器访问 `http://localhost:18090/wizard`，选择「修改密码」Tab。

### Q: 忘记密码了怎么办？

浏览器访问 `http://localhost:18090/wizard`，选择「忘记密码」Tab，输入你的邮箱和注册时的姓名验证身份，即可设置新密码。登录页面也有「忘记密码？」链接可以快速跳转。

### Q: 开机后 ai-monitor 没有自动启动？

重新执行全局安装：
```bash
ai-monitor.exe --global-install
```

### Q: 端口 18090 被其他程序占用了？

修改 config.json 中的 `port` 字段为其他端口（如 18091），然后重启 ai-monitor。

### Q: 我用的 AI 服务没有被监控到？

可以在 config.json 中添加 `extra_monitor_hosts`：
```json
{
  "extra_monitor_hosts": {
    "ai-api.mycompany.com": "internal-ai"
  }
}
```
